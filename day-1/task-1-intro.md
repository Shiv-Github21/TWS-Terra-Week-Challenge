# Understanding Infrastructure as Code (IaC)

## Objective

Understand the basics of Infrastructure as Code (IaC) and why Terraform is a popular choice.

---

## What is Infrastructure as Code (IaC)?

Infrastructure as Code (IaC) is the practice of managing and provisioning infrastructure using code instead of manually creating resources through a cloud console or graphical interface.

With IaC, we write configuration files that describe the infrastructure we want, and tools like Terraform automatically create and manage those resources.

---

## Why is IaC Important?

Infrastructure as Code provides several benefits:

- Automates infrastructure creation and management.
- Reduces manual errors.
- Ensures consistency across different environments.
- Makes infrastructure easy to version and track using Git.
- Enables faster deployments.
- Makes collaboration easier within teams.

---

## Why Terraform?

Terraform is an open-source Infrastructure as Code tool developed by HashiCorp.

It allows us to define infrastructure using a simple configuration language called **HCL (HashiCorp Configuration Language)**.

Terraform supports multiple cloud providers, including:

- AWS
- Microsoft Azure
- Google Cloud Platform (GCP)
- Oracle Cloud
- Kubernetes
- Docker

Because Terraform works with multiple providers, the same concepts can be used across different platforms.

---

## Key Benefits of Terraform

- Open Source
- Easy to Learn
- Multi-Cloud Support
- Declarative Configuration
- Infrastructure Automation
- State Management
- Strong Community Support
